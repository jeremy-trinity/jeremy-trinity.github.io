🐧 Linux. Imran Afzal · Complete Linux Training Course
Weeks 1-2 primary · Sections 1-8 this month
1
BOOTCAMP Section 1: Download and Install Linux
Install Ubuntu on an Azure VM, not a local VM. You practice on cloud from day one. Follow the section, but substitute VirtualBox with an Azure B1s VM (free tier). SSH into it from your local terminal after setup.
udemy.com → Complete Linux Training Course (Imran Afzal)
2
BOOTCAMP Section 2: System Access and File System
SSH access, file system hierarchy (/etc, /var, /home, /tmp), absolute vs relative paths. Practice every command in your Azure VM, not just watch.
3
BOOTCAMP Section 3: Linux Fundamentals
Navigation commands: ls, cd, pwd, mkdir, rmdir, cp, mv, rm, find. Hard links vs soft links. Everything you learn here: practice it live in your Azure VM before moving to the next section.
4
BOOTCAMP Section 4: File and Directory Permissions
chmod, chown, chgrp, umask, setuid/setgid/sticky bit. This is critical. You will use permission concepts every single day in production cloud work. Do not rush this section. Test every permission combination in your Azure VM.
5
BOOTCAMP Section 5: Linux File Maintenance Commands
tar, gzip, find with exec, locate, updatedb, df, du. Compress and extract a folder. Find all files modified in the last 24 hours. Practice in Azure VM.
6
BOOTCAMP Section 6: Text Processing: grep, awk, sed
grep with -r, -i, -n, regex patterns. awk for column extraction. sed for find-and-replace in files. These are critical for log analysis and will feed directly into KQL (Kusto Query Language) in Azure Monitor later. Practice on a real log file from your Azure VM (/var/log/syslog).
7
BOOTCAMP Section 7: Linux System Administration
User management (adduser, passwd, usermod, userdel), group management, sudo and sudoers file, cron jobs (crontab -e). Schedule a cron job that writes a timestamp to a file every minute, verify it works.
8
BOOTCAMP Section 8: Boot Process
BIOS/UEFI → bootloader (GRUB) → kernel → init system (systemd) → runlevels/targets. Understand the concept and order. You do not need to memorise every detail. This builds systems thinking that feeds Kubernetes later (containers also have startup sequences).
9
LAB OverTheWire Bandit. Levels 0 through 15
Do 2-3 Bandit levels per study day alongside the Udemy sections. This is a Linux security wargame where each level requires finding a password using Linux commands. By level 15 you will navigate the command line automatically without thinking. Do levels 0-15 across Month 1. Levels 16-33 are optional in Month 2.
overthewire.org/wargames/bandit

🌐 Networking. David Bombal · Complete Networking Fundamentals Course (targeted sections only)
Weeks 1-2 parallel · ~6-8 hours total across months 1-2
10
BOOTCAMP IP Addressing section. IPv4, CIDR, subnetting
This is the most critical networking section for Azure. Every Azure VNet address space and subnet calculation uses CIDR. Practice subnetting: given 10.0.0.0/16, carve out 4 subnets of /24. Understand why /24 gives you 254 usable hosts. This directly feeds Azure VNet design in Month 3.
udemy.com → Complete Networking Fundamentals Course (David Bombal)
11
BOOTCAMP DNS section, records, resolution, forwarders
A records, CNAME, MX, NS, TTL. How a DNS query resolves from root servers down. This feeds Azure DNS zones and Private DNS zones in Phase 2. Understand what a forwarder is. Azure Private DNS uses conditional forwarders for hybrid setups.
12
BOOTCAMP Firewalls and ACLs section, rule ordering
How firewall rules are evaluated in order (first match wins). Stateful vs stateless firewalls. This directly feeds Azure NSG (Network Security Group) rule priority. NSGs use the exact same first-match logic. A common AZ-104 exam question is about NSG rule priority.
13
BOOTCAMP Routing section, static and dynamic routing basics
Default gateway, routing tables, static routes. This feeds Azure User Defined Routes (UDRs), how you force traffic through an NVA (Network Virtual Appliance) or Azure Firewall. You do not need OSPF or BGP depth at this stage.
14
BOOTCAMP TCP/IP section, ports, protocols, connection establishment
TCP 3-way handshake, common ports (22 SSH, 80 HTTP, 443 HTTPS, 3389 RDP, 3306 MySQL, 5432 PostgreSQL). This feeds NSG rule writing. You need to know which port to open for which service. Also feeds security group troubleshooting.
